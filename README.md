# LoLMatchAnalyzer

## 1. 概要
LoLMatchAnalyzer は、Riot Games API を利用して League of Legends の試合データを収集・蓄積・分析する Python アプリケーションです。

本ソフトウェアは、自身の試合データを継続的に収集し、統計情報の可視化やプレイ傾向の分析を行うことで、プレイヤーの上達を支援することを目的としています。

また、将来的にはAIを活用した試合レビューや改善点の提案など、データ分析に基づくコーチング機能の実装も視野に入れています。

## 2. 開発目的
League of Legendsでは、試合結果や基本的な戦績はゲームクライアント上で確認できますが、長期間にわたるプレイ傾向の分析や、自身の成長を定量的に評価することは容易ではありません。

本プロジェクトでは、Riot Games APIを利用して試合データを継続的に収集・蓄積し、統計分析や可視化を行うことで、プレイヤー自身の課題や成長を客観的に把握できる環境を構築することを目的としています。

また、本ソフトウェアは分析基盤として設計し、将来的にはAIを活用した試合レビューやプレイ改善支援機能の実装も視野に入れています。

## 3. 主な機能
本ソフトウェアでは、以下の機能を提供する予定です。

* Riot Games APIを利用した試合データの取得
* 試合データのローカルデータベースへの保存
* プレイヤー情報およびランク情報の取得
* チャンピオン別・レーン別の戦績分析
* CS、KDA、ダメージなど各種統計情報の分析
* グラフによる分析結果の可視化
* CSV等へのデータ出力
* （将来実装予定）AIによる試合レビューおよび改善提案

## 4. ソフトウェアアーキテクチャ

本プロジェクトでは、保守性・拡張性・再利用性を考慮し、**レイヤードアーキテクチャ（Layered Architecture）**を採用します。

各モジュールは責務ごとに分割し、モジュール間の依存関係を最小限に抑えることで、機能追加や仕様変更に柔軟に対応できる構成を目指します。

また、単一責務の原則（Single Responsibility Principle）を設計指針とし、API通信・データベース・分析処理などを明確に分離します。

### 4.1 アーキテクチャ概要

本ソフトウェアは、以下のレイヤーで構成されます。

```text
┌───────────────────────┐
│        main.py        │
└──────────┬────────────┘
           │
           ▼
┌───────────────────────┐
│        Setup          │
└──────────┬────────────┘
           │
           ▼
┌───────────────────────┐
│    Configuration      │
└──────────┬────────────┘
           │
           ▼
┌─────────────────────────────────────────────┐
│                 Application                 │
├──────────────┬──────────────┬───────────────┤
│ Riot API     │ Database     │ Analysis      │
└──────────────┴──────────────┴───────────────┘
           │
           ▼
┌───────────────────────┐
│        Export         │
└───────────────────────┘
```

各レイヤーは上位レイヤーからのみ利用され、下位レイヤーは上位レイヤーへ依存しません。

### 4.2 各モジュールの責務

| モジュール             | 主な責務                                |
| ----------------- | ----------------------------------- |
| **Main**          | アプリケーションのエントリーポイント                  |
| **Setup**         | 初回起動時のセットアップ、環境構築                   |
| **Configuration** | `.env`・`settings.yaml` の読み込みおよび設定管理 |
| **Riot API**      | Riot Games APIとの通信およびレスポンス取得        |
| **Database**      | SQLiteへのデータ保存・取得・更新                 |
| **Analysis**      | 試合データの分析および統計処理                     |
| **Export**        | CSV・Excel・グラフなどへの出力                 |

各モジュールは単一の責務のみを担当し、他モジュールの内部実装へ直接依存しない設計とします。

### 4.3 設計方針

本プロジェクトでは、以下の設計方針に従って開発を行います。

* レイヤードアーキテクチャを採用し、責務を明確に分離する。
* 単一責務の原則（SRP）を遵守する。
* API通信・データベース・分析処理を独立したモジュールとして実装する。
* 設定情報と機密情報を分離し、安全な運用を実現する。
* モジュール間の依存関係を最小限に抑え、高い保守性を維持する。
* 将来的なGUI化、AI分析機能、外部データベースへの対応などを考慮した拡張性の高い構成とする。

ソフトウェアアーキテクチャの詳細設計については、`docs/architecture.md` に記載します。


## 5. ディレクトリ構成

本プロジェクトでは、ソースコード・設定ファイル・データ・ドキュメントを責務ごとに分離し、保守性および拡張性を考慮したディレクトリ構成を採用します。

```text
LoLMatchAnalyzer/
│
├── config/                  # アプリケーション設定
│   └── settings.yaml
│
├── data/                    # データベース・取得データ
│
├── docs/                    # 設計書・仕様書
│
├── logs/                    # ログファイル
│
├── src/                     # アプリケーション本体
│   ├── analysis/            # 試合データ分析
│   ├── common/              # 共通処理
│   ├── configuration/       # 設定管理
│   ├── database/            # データベースアクセス
│   ├── export/              # CSV・Excel等への出力
│   ├── riot_api/            # Riot Games API通信
│   └── setup/               # 初期設定・セットアップ
│
├── tests/                   # テストコード
│
├── .env                     # ユーザー固有設定（Git管理対象外）
├── .env.sample              # .env作成用テンプレート
├── .gitignore
├── main.py                  # エントリーポイント
├── README.md
├── requirements.txt
└── CHANGELOG.md
```

### ディレクトリ構成方針

* `src/` 配下には、アプリケーションのソースコードのみを配置します。
* `config/` には、Gitで管理する設定ファイルを配置します。
* `.env` には、APIキーやユーザー情報などの機密情報を保存し、Git管理対象外とします。
* `data/` には、SQLiteデータベースや取得データなど、実行時に生成されるデータを保存します。
* `docs/` には、READMEに記載しない詳細設計書や仕様書を配置します。
* `tests/` には、単体テスト・結合テストなどのテストコードを配置します。
* `logs/` には、アプリケーションのログファイルを出力します。

今後機能追加を行う際も、既存ディレクトリの責務を変更せず、必要に応じて新しいモジュールを追加する方針とします。
